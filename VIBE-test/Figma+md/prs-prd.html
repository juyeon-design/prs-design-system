개요
정의

PRS는 흩어진 제품 히스토리와 기획 맥락을 연결해, 누구나 빠르게 제품을 이해하고 이어서 판단할 수 있도록 돕는 제품 맥락 관리 서비스이다.

목표

흩어진 정보를 하나의 제품 맥락 안에서 연결함으로써 PO, 기획자, 개발자, 제품 리더가 같은 기준과 배경을 바탕으로 협업할 수 있게 한다.

또한 PRD 작성, 문서 탐색, 히스토리 확인, 승인 요청 등 기획 과정에서 반복적으로 발생하는 업무를 AI와 함께 처리하여 기획 생산성을 높이고, 기획 의도와 개발 결과물이 어긋나지 않도록 추적한다.

궁극적으로 PRS는 제품이 왜 만들어졌고, 어떤 방향으로 발전해야 하며, 현재 실행 결과가 그 방향과 잘 연결되어 있는지를 확인함으로써 제품의 상품성을 높이는 것을 목적으로 한다.

배경

현재 제품 관련 정보는 Confluence, Jira, Linear, 문서 파일, 회의 기록 등 여러 도구에 분산되어 있다. 이로 인해 제품을 새로 맡은 담당자나 협업자는 제품의 현재 상태와 과거 판단 근거를 파악하기 위해 여러 도구를 이동해야 한다. 또한 기획 의도와 개발 결과물 사이의 연결이 약해져, 시간이 지나면 왜 그렇게 만들었는지 알기 어려운 문제가 발생한다.

페인포인트
문제 정의
  • 1제품별 기획 문서와 히스토리가 흩어져 있어 맥락 파악 비용이 크다.
  • 2신규 담당자나 협업자가 제품을 이해하는 데 시간이 오래 걸린다.
  • 3기존 논의와 판단 근거를 찾기 어려워 중복 논의가 발생한다.
  • 4기획 의도가 개발-운영 과정에서 손상 없이 이어지는지 추적하기 어렵다.
해결 방안
  • 1제품 맥락 연결 : 흩어진 문서, 이슈, 회의, 히스토리를 제품 단위로 연결해 필요한 맥락을 빠르게 찾을 수 있게 한다.
  • 2기획 업무 생산성 향상 : PRD 작성, 문서 탐색, 히스토리 확인, 승인 요청 과정을 AI와 함께 처리해 기획 업무 시간을 줄인다.
  • 3상품성 관리 : 기획 의도와 개발 결과물이 연결되어 있는지 추적하고, 시장 방향과 제품 방향이 어긋나지 않도록 관리한다.
주요 기능
  • 1PRS 홈 : 전사 제품들을 한 곳에서 조망하고 필요한 제품 홈으로 빠르게 진입한다.
  • 2제품 홈 : 제품의 목적, 담당 조직, 리더, 현재 상태, PRD 요약, 최근 히스토리를 확인한다.
  • 3PRD 작성 및 관리 : 신규 PRD를 작성하고 기존 PRD를 수정하며, 문서 버전과 승인 상태를 관리한다.
  • 4AI 채팅 : 제품 문서, 이슈, 히스토리를 기반으로 질문에 답하고, 기획 문서 작성과 수정을 지원한다.
  • 5이슈 연동 : Jira 이슈를 제품 맥락과 연결해 기획 의도와 개발 실행 흐름을 함께 확인한다.
  • 6승인 요청 : 작성된 문서를 게시하고 관리자 또는 제품 리더에게 승인 요청을 보낸다.
고객 및 시나리오
주요 사용자
  • 1PO / 기획자 — PRD 작성, 기능 구조 설계, 승인 요청
  • 2개발자 — 이슈 확인, 기획 의도 추적, 변경 로그 조회
  • 3제품 리더 — 전체 맥락 파악, 버전 승인, 우선순위 조정
핵심 시나리오
  • 1신규 PO가 기존 제품의 맥락을 빠르게 파악하고 PRD를 이어서 작성한다.
  • 2기획자가 AI와 함께 PRD 초안을 생성하고 이슈로 분해하여 개발팀에 전달한다.
  • 3제품 리더가 분기 회고 시 기획 의도 대비 개발 결과를 추적한다.
성공 지표 및 분석
핵심 지표
  • 1PRD 작성 소요 시간 30% 단축 (AI 자동 생성 도입 후)
  • 2이슈-PRD 연결율 80% 이상 유지
  • 3팀 내 기획 맥락 공유 만족도 4.0/5.0 이상 (분기별 설문)
측정 방식
  • 1PRD 생성·수정 이벤트 로그 기반 소요 시간 집계
  • 2이슈-PRD 링크 연결 여부 DB 집계
  • 3분기별 팀 내 설문 (5점 척도)
