개요
정의

PRS는 흩어진 제품 히스토리와 기획 맥락을 연결해, 누구나 빠르게 제품을 이해하고 이어서 판단할 수 있도록 돕는다.

목표

흩어진 정보를 하나의 제품 맥락 안에서 연결함으로써 PO, 기획자, 개발자, 제품 리더가 같은 기준과 배경을 바탕으로 협업할 수 있게 한다.

또한 PRD 작성, 문서 탐색, 히스토리 확인, 승인 요청 등 기획 과정에서 반복적으로 발생하는 업무를 AI와 함께 처리하여 기획 생산성을 높이고, 기획 의도와 개발 결과물이 어긋나지 않도록 추적한다.

궁극적으로 PRS는 제품이 왜 만들어졌고, 어떤 방향으로 발전해야 하며, 현재 실행 결과가 그 방향과 잘 연결되어 있는지를 확인함으로써 제품의 상품성을 높이는 것을 목적으로 한다.

배경

현재 제품 관련 정보는 Confluence, Jira, Linear, 문서 파일, 회의 기록 등 여러 도구에 분산되어 있다. 이로 인해 제품의 현재 상태와 과거 판단 근거를 파악하기 위해 여러 도구를 이동해야 한다. 또한 기획 의도와 개발 결과물 사이의 간극이 생겨도 왜 그렇게 만들었는지 알기 어려운 문제가 발생한다.

페인포인트
문제 정의
  • 제품 기획 → 이슈 관리 → 리뷰 단계에서 맥락이 단절되어 정보 전달 오류 발생
  • 기획 문서(PRD) 작성에 과도한 반복 노동이 발생하고 최신화가 어려움
  • AI 기반 문서 자동화 부재로 경쟁사 대비 생산성 격차 증가
제품 범위 정의
  • 프로젝트 홈: 버전 현황 · 변경 로그 · PRD 미리보기 통합 대시보드
  • 기획 문서: AI 기반 PRD 자동 생성 + 노드뷰 기능 구조 설계
  • 이슈 목록: Epic / Story / SubTask 계층형 트리 이슈 트래킹
  • 프로젝트 설정: 멤버 관리 · 권한 · 연동 설정
주요 기능
  • AI 에이전트: 프로젝트 맥락을 수집·분석하여 PRD 초안을 섹션별로 자동 생성
  • 노드뷰: 기능 구조를 시각적 트리로 설계 (Root → L1 → Leaf, 최대 3뎁스)
  • 버전 관리: 문서 변경 이력 추적 · 버전 비교 · 롤백 지원
  • 승인 워크플로우: 검토 요청 · 코멘트 · 승인/반려 이력 관리
  • 이슈 연동: PRD 기반 이슈 자동 생성 · 상태 추적 · 버전 태깅
  • 싱크 설정: 외부 Jira, Linear 등과 이슈 상태 자동 동기화 처리
고객 및 시나리오
주요 사용자
  • PO / 기획자 — PRD 작성, 기능 구조 설계, 승인 요청
  • 개발자 — 이슈 확인, 기획 의도 추적, 변경 로그 조회
  • 제품 리더 — 전체 맥락 파악, 버전 승인, 우선순위 조정
성공 지표 및 분석
핵심 지표
  • PRD 작성 소요 시간 30% 단축 (AI 자동 생성 도입 후)
  • 이슈-PRD 연결율 80% 이상 유지
  • 팀 내 기획 맥락 공유 만족도 4.0/5.0 이상 (분기별 설문)
