# Empty State

Applies to all screen types — not a standalone screen type.

## Use When
- Data does not exist yet (initial / onboarding state)
- Search or filter returns no results
- Permission or configuration issue prevents items from showing

## Rules
- User must immediately understand why it is empty.
- Provide a next action when possible.
- "No search results" and "no data yet" require different messages and CTAs — do not reuse copy.

## Required Elements
- Illustration: 160×160px, centered
- Short title (Body1)
- Supporting description (Body2)
- Primary action button — if applicable
- Reset action — if caused by search or filter

## AI CTA Rule
- AI-related CTA: use `✦` prefix — e.g. `✦ 기능 설계하기`
- Non-AI CTA: Primary Button without prefix
